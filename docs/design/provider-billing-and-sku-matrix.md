# Provider-native Billing, SKU & Feature Matrix

This catalog captures the native billing, SKU, and runtime feature profile of each underlying sandbox/cloud provider. It answers the same questions for every provider:

- How does it charge?
- Does it support custom Docker images?
- Does it support snapshotting (disk/state capture)?
- Does it support memory / live-state snapshotting?
- What SKUs, sizes, and templates are available?

Prices are taken from each vendor's public pricing page as of 2026-07-16. Where no public pricing is available, the row is marked *not documented*. The focus is on the provider's own platform and API capabilities, not on how any SDK wraps them.

## Legend

- **Custom Docker images** — whether the provider can boot a sandbox from a user-supplied container image or Dockerfile at the platform level.
- **Disk / state snapshot** — whether a running sandbox's filesystem/state can be captured and later restored.
- **Memory snapshot / live state** — whether the provider preserves live RAM / process state across pause/resume/hibernate/fork, as opposed to only persisting disk.
- **SKUs / resource sizes** — the instance sizes, resource fields, or templates available at create time.
